# Report Writing Skill

## Report Structure

Follow this 7-section structure for all business analysis reports:

### Section 1: Title & Metadata
- Report title reflecting the analysis scope
- Date of generation
- Data source description
- Analysis parameters used

### Section 2: Executive Summary
- 5-7 bullet points covering the most critical findings
- Each bullet: one insight + one data point + one implication
- Maximum 200 words total

### Section 3: Data Overview
- Dataset description (source, time range, granularity)
- Key dimensions (categories, regions, products)
- Data quality notes (completeness, known issues)

### Section 4: Key Findings
- Organized by business theme (revenue, profitability, customer satisfaction, etc.)
- Each finding: context → data → interpretation → implication
- Include supporting metrics in parentheses

### Section 5: Detailed Analysis
- Deep-dive into each major finding
- Include segmented analysis (by region, category, time period)
- Reference any charts or visualizations created
- Compare against benchmarks where available

### Section 6: Recommendations
- Prioritized list of 3-5 actionable recommendations
- Each recommendation: action + expected impact + implementation difficulty
- Short-term (immediate) vs long-term (strategic) distinction

### Section 7: Appendix
- Full statistical tables
- Methodology notes
- Data dictionary
- Chart references

## Formatting Rules

- Use clear, professional language
- Avoid jargon; explain technical terms when first used
- Round numbers to 2 significant figures unless precision matters
- Use bullet points for lists of 3+ items
- Bold key metrics and figures for scanability
- Each paragraph: minimum 3 sentences

## Visual References

When including charts, reference them as:
- "See Figure 1: [Description]" in the text
- Place chart descriptions in the Appendix with full metadata
